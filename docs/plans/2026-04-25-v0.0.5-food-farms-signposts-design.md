# v0.0.5 Food, Farms, and Signposts Design

## Context

**Prompt:** “/start v0.0.5 food resource … show food as resource (sprite #98). Additionally food can be replenished in farms (tile/sprite #38) which need to be promoted from cosmetic to an event tile. Also signposts should be changed to point to the nearest PoI (castle or farm). Add some animation for the food depleting/being replenished (flash +N / -N). Make farm/revisit wording more magical, chosen from a small set. Signposts should use named farms.”

**Reasoning:** The current prototype has deterministic worldgen, a pure reducer, and a read-only renderer. This slice adds one resource and one PoI event (Farm) while preserving determinism and keeping systems small and serializable (no functions/classes in state).

---

## Overview

v0.0.5 introduces:

- **Food resource**: starts at **10**, costs **1 per MOVE**, never below **0**.
- **3 Farm PoIs**: placed as specials; stepping onto a ready farm yields **3..10** food (deterministic), then the farm enters a **3-move cooldown**.
- **Signposts**: when stepped on, they point to the **nearest Point of Interest** (PoI): **the Castle** or **a named Farm**.
- **Food delta animation**: a small, non-blocking float/flash of **`-1`** (move cost) and **`+N`** (farm harvest) rendered near the food value.
- **Flavor text pools**: farm harvest and “empty/cooldown” visits pick a line from curated sets in a deterministic, non-abusable way.

Non-goals for v0.0.5: gold/army, combat, towns/shops, fog-of-war/visited tracking (beyond farm cooldown).

## Constants and tunables

To keep iteration cheap, all user-facing strings and most knobs should live in `src/core/constants.ts` (or a `src/core/flavor.ts` if `constants.ts` gets too crowded):

- `INITIAL_FOOD = 10`
- `FOOD_MOVE_COST = 1`
- `FOOD_WARNING_THRESHOLD = 5` (render food amount in a warning color when `food < threshold`)
- `FARM_COUNT = 3`
- `FARM_COOLDOWN_MOVES = 3`
- `FOOD_SPRITE_ID = 98` (16×16; render with `w=2,h=2`)
- `FARM_NAME_POOL = [...]`
- `FARM_HARVEST_LINES = [...]`
- `FARM_REVISIT_LINES = [...]`

This lets us tune pacing and tone without touching reducer logic.

## Data model changes

### World

Add Farm PoIs as first-class specials alongside the castle.

- `world.farms: { position: Vec2; name: string }[]` (length **3**, stable order)
  - Names are chosen deterministically from a fixed pool, **without repeats** in a world.
  - If the pool ever depletes (future-proofing), fall back to the name `"A Farm"`.

Tile representation remains `world.tiles[y][x]` with numeric tile IDs, but:

- Tile **#38** is removed from the base terrain cosmetic pool and instead used only for farm specials.

### State

Add resources and farm cooldown bookkeeping:

- `state.resources.food: number`
- `state.resources.farmNextReadyStep: number[]` aligned with `world.farms` (length 3)
  - “Ready” when `state.run.stepCount >= farmNextReadyStep[i]`.
  - After successful harvest on farm `i`: set to `state.run.stepCount + 3`.

### UI animation

Extend `state.ui.anim.active` to allow a new non-blocking anim kind:

- `kind: 'foodDelta'`
- `durationFrames`: short (e.g. 20–30)
- `blocksInput: false`
- `params: { delta: number }` where `delta` is `-1` for move-cost and `+N` for harvest payout.

Multiple `foodDelta` anims may be enqueued in the same MOVE resolution (e.g. `-1` and `+7`).

## World generation (deterministic)

Starting point is `generateWorld(seed)` which currently:

1. Builds base terrain via noise → walkable cosmetic tiles
2. Places **1 Castle** tile
3. Places **SIGNPOST_COUNT** signposts
4. Picks random start position

Update to:

1. Build base terrain (remove tile #38 from the cosmetic tile set)
2. Place **1 Castle**
3. Place **3 Farms** (tile #38), distinct from castle and from each other
4. Assign **3 unique farm names** deterministically from a fixed name pool
5. Place **SIGNPOST_COUNT** signposts, distinct from castle and farms
6. Pick start position

Farm name pool (fixed order):

- The_Oast
- Burnt_Acre
- Greyfield
- Hobs_Reach
- The_Stemming
- Fallow_End
- Cotters_Rise

Display names should render with spaces/apostrophes (e.g. `Hob's Reach`) but storage can be plain strings.

Determinism requirement:

- Same seed → identical `world.tiles`, `world.castlePosition`, `world.farms`, and `startPosition`.

## Reducer rules

### NEW_RUN(seed)

Initialize:

- `resources.food = 10`
- `resources.farmNextReadyStep = [0, 0, 0]`
- All other state as today.

### MOVE(dx, dy)

On every MOVE:

- Increment `run.stepCount` by 1
- Apply food move cost: `food = max(0, food - 1)`
- Enqueue a `foodDelta` anim of `-1` **only if food actually decreases** (i.e. previous food > 0)

Then, based on the destination tile:

- **Farm tile (#38)**
  - Identify which farm index it is (by position match in `world.farms`)
  - If farm is ready:
    - Harvest `gain ∈ [3..10]` deterministically
    - Apply: `food += gain`
    - Enqueue `foodDelta` anim of `+gain`
    - Set cooldown: `farmNextReadyStep[i] = run.stepCount + 3`
    - Message: one “harvest” line (below), plus an optional short second line like `(+<gain> food)`
  - If not ready:
    - No gain; message from “revisit” pool, plus an optional short line like `(return in <N> moves)`

- **Signpost tile (#42)**
  - Message points to nearest PoI (castle or named farm), below

- **Castle / other terrain**
  - Existing behavior as today (castle found message; terrain messages)

### Flavor text pools (deterministic, non-abusable)

We want variety without letting “spam revisits” reroll future harvest results.

- **Harvest success** (consumes RNG; already needs RNG for `gain` anyway):
  - Choose a line from:
    - “Someone left in a hurry. The stores are still full.”
    - “The cellar is cold and deep. You help yourself.”
    - “Enough here to keep moving. You take what you need.”
    - “Unharvested, but not unwelcome. You gather what you can.”
    - “The farmer is long gone. The food remains.”
  - RNG consumption order: pick `gain` first, then pick message index.

- **Revisit / cooldown** (does **not** consume RNG):
  - Choose a line from:
    - “You already took what there was.”
    - “The stores are empty now. Come back later.”
    - “Nothing left here. It will regrow in time.”
  - Deterministic selection rule (exact):
    - Let `k = (world.seed + farmIndex * 7 + state.run.stepCount) | 0`
    - `revisitLineIndex = ((k % FARM_REVISIT_LINES.length) + FARM_REVISIT_LINES.length) % FARM_REVISIT_LINES.length`
    - Use `FARM_REVISIT_LINES[revisitLineIndex]`
  - This varies over time but never advances RNG, so it cannot perturb future harvest yields.

## Signposts: nearest PoI (Castle or named Farm)

When stepped onto, compute the nearest PoI from the current position (the signpost tile):

- Candidate PoIs:
  - Castle at `world.castlePosition`
  - Each farm at `world.farms[i].position`
- Distance: torus-shortest Manhattan distance using existing `torusDelta()` and `manhattan()`
- Direction label: existing `dirLabel(dx, dy)` using the same deltas
- Tie-breaks (deterministic):
  - Prefer **Castle** over Farm
  - If multiple farms tie, pick the earliest `world.farms` order

Message format (two-line to fit `LORE_MAX_CHARS_PER_LINE=19`):

- Farm:
  - Line 1: `<FarmName> Farm`
  - Line 2: `<DIR>, <D> leagues away.`
- Castle:
  - Line 1: `The Castle`
  - Line 2: `<DIR>, <D> leagues away.`

## Rendering / UX notes

### Food status line

Add a 4th status line in the left panel:

- Icon: sprite **#98** (16×16)
- Value: current food number
- Render rule: when `food < FOOD_WARNING_THRESHOLD`, render the number in a warning color (otherwise normal text color).

### Food delta animation

At render time, find active `foodDelta` anims and render a small transient text near the food value:

- `-1` in a “warning” color
- `+N` in a “good” color
- Optional tiny upward drift or blink over ~20–30 frames
- Non-blocking (does not prevent movement)

If both `-1` and `+N` are present for the same move, show both (stacked or offset) so the numbers match the player’s mental model.

## Trade-offs considered

- **Farm identity in world vs derived from tiles**: storing `world.farms[]` (positions + names) makes cooldown, naming, and signpost targeting explicit and testable; it avoids “scan tiles every frame” patterns and keeps reducer logic simple.
- **Revisit flavor without RNG**: revisit text uses a deterministic index function so it can vary without consuming RNG, preserving the “no revisit can affect future harvest RNG” property.
- **Food delta as `ui.anim` kind**: keeps the reducer as the single place where deltas are produced, and the renderer as the only place that visualizes them; avoids hidden UI state in the renderer.

## Acceptance Criteria

| # | Scenario | Expected result |
|---:|---|---|
| 1 | Start a new run | Food starts at **10**; farms exist in the world as specials |
| 2 | Move onto a normal tile | stepCount increments; food decreases by **1**; `-1` delta anim appears |
| 3 | Food cannot go negative | Food bottoms at **0** |
| 4 | Farms are placed | Exactly **3** farm tiles exist; they do not overlap castle/signposts; names are unique within the run |
| 5 | Step onto a ready farm | Food increases by **gain ∈ [3..10]**; `+gain` delta anim appears; farm enters cooldown; message is from harvest pool |
| 6 | Step onto a farm in cooldown | No food gain; message is from revisit pool; indicates remaining moves (optional) |
| 7 | Farm cooldown expires | After **3** moves since harvest, the same farm can pay out again |
| 8 | Signpost message | Points to nearest PoI (castle or farm) with correct dir+distance; format is two-line as specified |
| 9 | Determinism | Same seed + same click sequence yields identical outcomes, including farm placement/names and farm gains/messages |
