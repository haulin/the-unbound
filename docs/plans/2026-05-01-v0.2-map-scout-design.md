# v0.2 — Map & Scout (Design)

## Context

**Prompt:** `/start v0.2 see docs/backlog.md docs/the-unbound-learnings.md docs/lore-and-tone.md`

v0.2 ships two player-facing additions that build directly on v0.1 “Lost”:

- **An in‑game map** (not the debug minimap), designed to **respect being lost** (no free player position while lost).
- **A Scout** that is **acquirable during the run** in v0.2 (acquisition via Camps, since Towns are deferred).

**Constraints:**

- **Pure reducer + serializable state**.
- **Lost remains meaningful** (information-state mechanic).
- **Clarity**: stable button layout + lore feedback beats “buttons disappearing”.
- **Tone**: quiet, practical.

**Note (sprite-sheet pressure):** button/icon sprite IDs are getting tight on sheet 1. v0.2 uses specific IDs but expects a possible rearrange post‑phase.

---

## Locked decisions (quick list)

- **Map button**: right-grid **top-right corner**; sprite **`#138`**.
- **Camp encounter**: modal encounter when stepping onto a camp; static buttons:
  - **Search camp**: sprite **`#110`**
  - **Hire Scout**: sprite **`#142`**
  - **Leave**: reuse existing “return” icon (same visual verb as combat leave)
- **Scout**: costs **5 food**, icon **`#108`**, shows in global HUD when owned.
- **Scout effects**:
  - Lost chance on woods/swamp is halved (**floor**).
  - When **oriented**, map reveals **Farms/Camps/Henges** globally.
  - Gate/Locksmith remain gated by mapping (even with Scout).
- **Map rendering** (rolling map):
  - Fixed viewport centered on the player (torus-wrapped); player marker is always in the center (does not reveal global position when lost).
  - Landmarks-only letters (`F/C/H/G/L`) at 6px pitch.
  - Tile background uses sprite `#135` (colorkey 0) stamped per cell.
  - Player marker uses outline sprite `#134` (colorkey 0).
- **Map availability**: Map toggle is available everywhere (including during **combat** and **camp** encounters). Closing the map restores the previous left-panel mode (and restores the prior lore message if it was not overwritten).
- **Map toggle lore**: opening the map temporarily overrides the lore message; closing restores the prior message if unchanged.
- **Run memory**: `run.path: Array<{ pos; isMapped }>` + `lostBufferStartIndex` to support:
  - buffer while lost,
  - backfill when re-oriented,
  - discard previous buffer on repeat teleport while still lost (by overwriting `lostBufferStartIndex`).
- **Camp does not rescue hunger death**: if entry hunger reduces army to 0, you die immediately (camp menu does not open).

## Overview (player-visible behavior)

### In‑game map

- The map is a left-panel mode toggled by the Map button.
- The map shows **only landmark markers**, not terrain.
- The map is a **rolling view** centered on the player; the player marker is always shown at the center.
- When the player is **lost**, the map still shows a local fragment of what you have encountered since the latest time you were pulled off course.

### Camps (contextual encounter)

- Stepping onto a camp enters a Camp encounter (like combat remaps controls).
- Camp encounter keeps a stable three-verb cross: Search camp / Hire Scout / Leave.
- Camp preview shows up to three stacked lines (omit non-applicable):
  - reinforcements (+food, +army) if camp is ready,
  - scout cost line if scout not owned.

## Data model

### Resources

```ts
resources: {
  food: number
  armySize: number
  hasBronzeKey: boolean
  hasScout: boolean // NEW
}
```

### Run travel history + mappedness

```ts
run: {
  stepCount: number
  hasWon: boolean
  isGameOver: boolean
  knowsPosition: boolean

  path: Array<{ pos: Vec2; isMapped: boolean }> // NEW
  lostBufferStartIndex: number | null           // NEW
}
```

**Update rules:**

- On MOVE completion, append `{ pos: finalPlayerPos, isMapped: false }`.
- If `finalKnowsPosition === true`:
  - mark the newest entry `isMapped = true`.
  - if `lostBufferStartIndex != null`, mark entries `[lostBufferStartIndex..last]` mapped and then clear it.
- On lost teleport:
  - force `knowsPosition = false`.
  - set `lostBufferStartIndex = lastIndex` (overwrite if already set).

### Encounters + actions

Add a camp encounter variant.

```ts
type Encounter =
  | CombatEncounter
  | CampEncounter
```

```ts
type CampEncounter = {
  kind: 'camp'
  sourceKind: 'camp'
  sourceCellId: number
  restoreMessage: string // the tile-enter message to restore after leaving the camp encounter
}
```

New actions (names illustrative; exact constants can vary):

- `TOGGLE_MAP`
- `CAMP_SEARCH`
- `CAMP_HIRE_SCOUT`
- `CAMP_LEAVE`

**Turn semantics:** camp actions do **not** increment `run.stepCount` (MOVE does). This keeps cooldown and deterministic offers stable.

### Defaults / normalization

When constructing state from legacy tests or future save/load, default missing fields as:

- `resources.hasScout = false`
- `run.path = []`
- `run.lostBufferStartIndex = null`

## Algorithms

### Deterministic camp offer (preview matches outcome)

Camp offer is computed without consuming `world.rngState`.

- Ready predicate: `run.stepCount >= camp.nextReadyStep`.
- Food offer: `foodGain = CAMP_FOOD_GAIN` when ready else `0` (current constant is 2).
- Army offer (when ready):

```ts
armyGain = 1 + (hashSeedStepCell({ seed: world.seed, stepCount: run.stepCount, cellId: camp.id }) % 2)
```

### Scout halves lost chance (floor)

When rolling the tile event for woods/swamp, apply:

```ts
lostPct = hasScout ? Math.floor(lostPct / 2) : lostPct
```

## UI / rendering (TIC‑80)

### Right-grid button mapping

- Corners (meta):
  - top-left Goal
  - bottom-left Debug minimap (unchanged)
  - bottom-right Restart
  - top-right Map (new; sprite `#138`)

Camp encounter cross:

- West: Search camp (sprite `#110`)
- North: Hire Scout (sprite `#142`)
- East: Leave (reuse existing return arrow)
- Center: camp icon `#140` (no-op)
- South: disabled

### Camp preview plate (left panel)

- Layout: combat-style plate inside the 64×64 illustration block.
- Lines: up to 3, omit if not applicable:
  - Food icon + `+N` (only when ready; N is 2)
  - Army icon + `+N` (only when ready; N from deterministic formula)
  - Scout icon `#108` + `5` (only when Scout not owned)

### Map rendering

- Draw a centered 9×9 viewport (6px pitch) inside the 64×64 block (torus-wrapped around the player).
- Stamp sprite `#135` (colorkey 0) behind each 6×6 cell.
- Markers are letters:
  - `F` farm, `C` camp, `H` henge, `G` gate, `L` locksmith
- Colors:
  - markers: light grey
  - player outline: sprite `#134` (always shown at viewport center)

Marker visibility:

- Oriented (no Scout): show only landmarks whose coordinates appear in the **mapped** portion of `run.path`.
- Oriented (with Scout): always show `F/C/H`; show `G/L` only when mapped/visited.
- Lost: show only landmarks encountered since `lostBufferStartIndex` (local fragment; no global reveal).

## Rejected alternatives

- **Dynamic camp menus** (hide buttons when unavailable): rejected for discoverability and learnability.
- **Map shows player marker revealing global position while lost**: rejected (breaks v0.1 Lost).
- **Color-only map**: rejected (accessibility + ambiguity).
- **`cell.visited: boolean`**: rejected (weak for buffering/discard + future replay).

## Acceptance Specs (GWT)

;===============================================================
; Camps are always a contextual stop.
;===============================================================
GIVEN the player is traveling the overworld.

WHEN the player steps onto a camp.

THEN the player is shown camp options.

AND the player cannot continue traveling until they choose Leave.

;===============================================================
; Camp options are static (always present).
;===============================================================
GIVEN the player is at a camp.

THEN the camp options include Search camp, Hire Scout, and Leave.

;===============================================================
; Search camp can succeed or be empty (cooldown expressed as lore).
;===============================================================
GIVEN the player is at a camp.

WHEN the player chooses Search camp.

THEN either the camp provides reinforcements, OR an “empty camp” lore message is shown.

;===============================================================
; Searching twice on one visit yields empty the second time.
;===============================================================
GIVEN the player is at a camp.

WHEN the player chooses Search camp and receives reinforcements.

AND the player chooses Search camp again without leaving.

THEN an “empty camp” lore message is shown.

;===============================================================
; Hiring Scout at a camp.
;===============================================================
GIVEN the player is at a camp and does not have a Scout.

WHEN the player chooses Hire Scout with at least 5 food.

THEN the player pays 5 food.

AND the Scout joins the player.

;===============================================================
; Hiring Scout when already owned yields lore.
;===============================================================
GIVEN the player is at a camp and already has a Scout.

WHEN the player chooses Hire Scout.

THEN an “already have a scout” lore message is shown.

AND nothing else changes.

;===============================================================
; Hiring Scout without enough food yields lore.
;===============================================================
GIVEN the player is at a camp and does not have a Scout.

WHEN the player chooses Hire Scout with less than 5 food.

THEN a “not enough food” lore message is shown.

AND the Scout does not join.

;===============================================================
; Scout reveals landmarks on the in‑game map (but not gate/locksmith).
;===============================================================
GIVEN the player has hired the Scout.

WHEN the player opens the in‑game map.

THEN farms, camps, and henges are visible even if unvisited.

AND the gate and locksmith are not revealed by the Scout.

;===============================================================
; Map respects being lost.
;===============================================================
GIVEN the player is lost (their position is unknown).

WHEN the player opens the in‑game map.

THEN the map does not show the player’s current position marker.

;===============================================================
; Map shows player marker when oriented.
;===============================================================
GIVEN the player is oriented (their position is known).

WHEN the player opens the in‑game map.

THEN the map shows the player’s current position marker.

;===============================================================
; Scout reduces the chance of becoming lost.
;===============================================================
GIVEN the player has hired the Scout.

WHEN the player steps onto woods or swamp.

THEN the chance of being pulled off course is reduced by half (rounding down).

;===============================================================
; Route knowledge buffered while lost (discard buffer on repeat teleport).
;===============================================================
GIVEN the player is lost.

WHEN the player travels and later becomes oriented again.

THEN the map updates to reflect where the player has been since the most recent time they were pulled off course.

AND if the player is pulled off course again while already lost, they lose the buffered mapping progress from before that second teleport (but the run travel history is not erased).

;===============================================================
; Camp does not rescue hunger death on entry.
;===============================================================
GIVEN the player has no food and their army would drop to 0 on entry.

WHEN the player steps onto a camp.

THEN the run ends immediately.

AND the camp encounter does not open.

## Behavioral Impact

- **Modified behaviors:**
  - Camps change from “tile-enter auto effect” to “enter camp encounter; actions chosen via buttons”.
  - Camps no longer “save you” on entry if hunger would reduce army to 0.
  - Right-grid top-right corner becomes an active Map button (was disabled).
- **New behaviors:**
  - In‑game map (landmarks-only).
  - Scout acquisition + Scout status + Scout effect on lost chance.
  - Replay-ready run travel history with mapped/unmapped segments.
- **Existing tests requiring updates:**
  - Camp reducer tests that assume auto-recruit on MOVE onto camp.
  - Any tests building `resources` literals will need `hasScout: false`.
  - Any tests building `run` literals will need `path` and `lostBufferStartIndex` (or normalization defaults).
