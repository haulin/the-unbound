**Goal:** Ship v0.6 — re-flavor combat into Brigand / Goblin / Henge / Wyrm variants with per-variant round math, spawn, reward, and payment; add persistent henge bands; add recruit button; fix food-cap double-punishment; widen ambush spawn floor.

**Use case:** A player on a fresh seed walks into woods and fights goblins (food drop), into mountains and fights brigands (gold drop, harder math), reaches a henge and fights a band of armed men they can flee-and-return-to without losing the fight's progress, and at the end of a combat with ≤5 enemies left can recruit the survivors instead of finishing them off.

**Non-goals:**
- No henge tiers (lesser / standard / great) — deferred to v0.7.
- No 3:1 goblin/brigand territorial mix — pure split (woods → goblin, mountain → brigand) for v0.6; requires no variant-on-encounter migration.
- No combo breaker / pity timer / streak smoothing — sim shows goblin re-flavoring covers it.
- No "rich enemy" tag-multiplier — subsumed by reward scaling.
- No wyrm gold loot when wounded — keeps wyrm-blood as the unique prize.
- No new top-slot sprite — recruit slot reuses `SPRITES.inventory.gold` per wyrm precedent.
- No changes to `WOODS_LOST_PERCENT`, `MOUNTAIN_AMBUSH_PERCENT`, `SWAMP_LOST_PERCENT` — terrain hazard rates are out of scope.
- No swamp combat — swamp keeps its lost-only policy; `terrainHazardsMechanic.combatVariantByKind` has no `swamp` entry.
- No new ADR file — `docs/decisions/` infrastructure waived for v0.6 per design.

**Architecture:** Single architectural move + plug-ins. `MechanicDef.combatVariant` → `combatVariantByKind: Partial<Record<CellKind, CombatVariantConfig>>`. `CombatVariantConfig` gains `playerRollBonus`, `enemyRollBonus`, `spawnEnemy`, and a non-optional `payment` block with discriminated-kind eligibility. `CombatEncounter` gains `initialSpawn`. Each variant lives on its source mechanic def. Combat reducer reads variant fields directly; no new branching, just two extra constructor args to `resolveFightRound` and a four-way dispatch on eligibility kind in `reduceCombatPay`. Henge persistence adds `currentGroup: number | null` on `HengeCell`. Food cap helper renames + 5 callsite migration.

**Tech Stack:** TypeScript, Vitest, esbuild build to `the-unbound.js` (TIC-80 cart). No new runtime deps; sim script (`scripts/combat-sim.mjs`) is dev-only.

```ts
function reduceCombatPay(prevState: State): State {
  const enc = prevState.encounter
  if (!enc || enc.kind !== 'combat') return prevState
  const variant = combatVariantForEncounter(prevState)
  const eligibility = variant.payment.isEligible(enc, prevState.resources)

  if (eligibility !== 'ok') {
    const lines = variant.payment.failLines[eligibility] ?? []
    if (lines.length === 0) return prevState  // defensive: no lines = no message change
    const pick = RNG.createRunCopyRandom(prevState).advanceCursor(`combat.pay.${eligibility}`, lines)
    return {
      world: prevState.world,
      player: prevState.player,
      run: pick.nextState.run,
      resources: prevState.resources,
      encounter: enc,
      ui: { ...prevState.ui, message: pick.line || prevState.ui.message },
    }
  }

  // OK path: existing wyrm success flow generalised.
  const cost = variant.payment.computeCost(enc)
  const afterDeduct: Resources = { ...prevState.resources, gold: prevState.resources.gold - cost }
  const nextResources = variant.payment.onSuccess(afterDeduct, enc)
  const successPick = RNG.createRunCopyRandom(prevState).advanceCursor('combat.pay.success', variant.payment.successLines)
  const nextWorld = applyCombatResolved(prevState.world, enc.sourceCellId)
  // ... rest of the existing wyrm success path (animations, encounter close)
}
```

export function brigandRecruitCost(enc: CombatEncounter): number {
  return enc.enemyArmySize * enc.enemyArmySize
}

export function brigandRecruitEligibility(enc: CombatEncounter, res: Resources): EligibilityKind {
  if (enc.enemyArmySize > BRIGAND_RECRUIT_MAX_REMAINING) return 'tooMany'
  if (enc.enemyArmySize >= enc.initialSpawn) return 'notWounded'
  if (res.gold < brigandRecruitCost(enc)) return 'noFunds'
  return 'ok'
}

export function brigandRecruitLootScale(enc: CombatEncounter): number {
  if (enc.initialSpawn <= 0) return 0
  return (enc.initialSpawn - enc.enemyArmySize) / enc.initialSpawn
}

```ts
export function applyFoodCapOnGain(prev: Resources, next: Resources): Resources {
  if (next.food <= prev.food) return next  // shrink or no-change → never clamp
  const cap = next.armySize * FOOD_PER_TROOP
  return next.food > cap ? { ...next, food: cap } : next
}
```

(Two-argument shape captures intent: clamp only the *delta*, evaluated against the *new* cap. If both rise, the gain is clamped to whatever the new cap allows. If only food rises, capped against the new cap. If only army shrinks, returned unchanged.)

function rolledEnemySpawn(playerArmy: number, rngState: number): { enemyArmy: number; rngState: number } {
  const r = RNG.createStreamRandom(rngState)
  const min = Math.max(2, playerArmy - 2)
  const max = 2 * playerArmy
  const span = max - min + 1
  return { enemyArmy: min + r.intExclusive(span), rngState: r.cursor }
}
