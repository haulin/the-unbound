# v0.7 Terrain Payoff Implementation Plan

> **Design:** `docs/plans/2026-06-02-v0.7-terrain-payoff-design.md`

**Implementation stance:** Prefer fewer files and shared helpers over proliferating mechanic modules. Extend via new variant configs and table rows (open/closed). Refactor when the same pattern appears a third time — do not freeze layout if code review shows a better seam.

## Goal

Ship inline quiet-enter finds on swamp and mountain (primary + secondary loot, per-biome lore, delta flashes). Complete v0.6 variant file split and lore pass if still open before or as Chunk 1.

**Non-goals:** woods finds, modals, new sprites, camp local map, healer, find-rate tuning beyond constants table (playtest adjusts constants only).

**TDD:** enforce — acceptance file first for find paths. **ATDD:** enforce — 6 GWT specs in design (5 terrain find + 1 rainbow gold delta).

### As-built module layout (post-review)

| Concern | Location |
|---------|----------|
| Biome enter + combat variant | `defs/woods.ts`, `defs/swamp.ts`, `defs/mountain.ts` (three `MechanicDef`s) |
| Henge / wyrm variants | `defs/henge.ts`, `defs/wyrm.ts` |
| Combat reducer + types | `defs/combat.ts` |
| Terrain move-event + quiet find | `encounterHelpers.ts` (`resolveTerrainMove`, `rollTerrainFindPayout`) |
| Quiet-find config + rolls | Per-biome def (`SWAMP_QUIET_FIND`, …); generic `tryQuietFind` in encounterHelpers |
| Extra keyed draws on same tile step | `RNG.domainSalt(label)` + `keyedBoundedBase` |
| Gameplay constants (%, bases, noise) | `constants.ts` |

Removed during implementation: `terrainHazards.ts`, `terrainFind.ts`, `combatVariants.ts`.

## Chunk 1 — Combat encounter seam (prerequisite, discovery-driven)

**Intent:** Fewer files, shared helpers, open/closed. Plans are a guide — if the same enter/combat/open pattern appears twice, note it; on the third copy, extract. Do not add files per terrain kind unless a table-driven single mechanic cannot stay clear.

**1.1 Inventory combat openers** (read before moving code):

| Source | Opens combat via | Variant today |
|--------|------------------|---------------|
| Terrain woods/mountain | `rollMoveEvent` → `startCombatEncounter` | goblin / brigand in `combat.ts` |
| Terrain swamp | lost only (find lands here in Chunk 3) | — |
| Wyrm lair | `startCombatEncounter` + bled gate | `wyrm.ts` |
| Henge | cooldown / `currentGroup` → `startCombatEncounter` | `henge.ts` (+ brigand recruit helpers from `combat.ts`) |

**1.2 Target shape (as built):**

- `combat.ts` — encounter **actions** only (`reduceCombat*`), `CombatVariantConfig` type, `startCombatEncounter`, spawn helpers. No variant objects.
- Variant configs **colocated** with their mechanic: goblin → `woods.ts`, brigand (+ recruit helpers) → `mountain.ts`, henge → `henge.ts`, wyrm → `wyrm.ts`.
- `encounterHelpers.ts` — shared terrain move-event resolution and quiet-find payout (`resolveTerrainMove`, `rollTerrainFindPayout`). Henge/wyrm keep bespoke `onEnterTile`; they share combat open via `startCombatEncounter` only.
- `woods.ts` / `swamp.ts` / `mountain.ts` — one `MechanicDef` each; swamp/mountain pass `tryQuietFind: true` into `resolveTerrainMove`.
- `rng.ts` — `domainSalt(label)` when a second keyed numeric draw shares seed/step/cell; find lore via `perMoveLine`.

**1.3 Wyrm polish** (same chunk if touching variants): `onCombatClosed` outcome `'recruit'` vs `'paid'` cleanup per Polish backlog — only if already in the files you touch.

**1.4 v0.6 lore pass** — `lore.ts` index matches shipped combat (goblin/brigand/henge recruit, barkeep tips). No find lines yet.

**Verify:** `npm test`; determinism golden only if RNG draw order or MECHANICS order changes.
