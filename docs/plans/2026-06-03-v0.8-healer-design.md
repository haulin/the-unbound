## Overview

### Placement RNG (part of v0.8)

```text
seed
 Ōö£ŌöĆ place.terrain     ŌåÆ grid terrain only
 Ōö£ŌöĆ place.wyrm        ŌåÆ lair tile(s)
 Ōö£ŌöĆ place.locksmith   ŌåÆ (reads gate cell; own stream)
 Ōö£ŌöĆ place.gate
 Ōö£ŌöĆ place.farm
 Ōö£ŌöĆ place.camp
 Ōö£ŌöĆ place.town
 Ōö£ŌöĆ ŌĆ” each mechanic with placeWorld
 Ōö£ŌöĆ place.start       ŌåÆ player start; legacy `world.rngState` for run sim
 ŌööŌöĆ *.offers          ŌåÆ menus/prices only (forked per kind)
```

- **`placeWorld`** uses `placeFeatureFromSeed` / `placeNamedFeatureFromSeed` and returns incoming **`rngState` unchanged** (passthrough).
- **Rule:** no mechanic reads another mechanicŌĆÖs placement stream. Peer constraints (locksmith Ōåö gate) read **cells**, not shared dice order.
- **Adding a town** changes only `place.town` and `town.offers`; farms/camps/henges on the same seed keep prior tiles.

**Intentionally out of scope for v0.8:** combat round isolation (keyed per fight or per-encounter fork). Documented as **target** in learnings; implement in a follow-up slice.

### `buildOfferSets` (towns, camps, farms)

Shared helper (`mechanics/offerSets.ts`):

1. **Map pass** ŌĆö each `mustCover` offer appears on Ōēź1 PoI of that kind (random eligible PoI per offer, `*.offers` RNG stream).
2. **Backfill** ŌĆö each PoI gets **1ŌĆō3** distinct offers from its pool (RNG on `*.offers` domain stream).
3. **Constraints**
   - Ōēź1 offer per PoI (never leave-only).
   - Ōēź1 offer with `category !== 'companion_hire'` per PoI.

| PoI | Pool (v0.8) | mustCover (map-wide) |
|-----|-------------|----------------------|
| Town | food, troops, rumor, hireScout, hireHealer | food, troops, rumor, hireScout, hireHealer |
| Camp | search, hireScout | search, hireScout |
| Farm | buyFood, hireMule | buyFood, hireMule |

**Categories:** `economy` (food/troops/rumor), `companion_hire` (hire*), `camp_action` (search).

**`TOWN_COUNT = 3`.** Drop scout-on-town-0 hack.

**Layout:** offers fill modal slots in order **left ŌåÆ top ŌåÆ bottom** (`offersToGridLayout`); offer array order from `buildOfferSets` backfill. No per-PoI priority tables.

**Farm:** every farm includes `buyFood` (`requiredOnEveryPoi`).

**`buildOfferSets` capacity:** needs `poiCount ├Ś maxOffers Ōēź |mustCover|` and at least one economy offer in the pool. Each PoI ends with Ōēż `maxOffers` distinct offers and Ōēź1 non-hire (may swap out a hire when full).

### Companion hires

Pillar: [`the-unbound-learnings.md` ┬¦ Teach through interaction, not absence](./the-unbound-learnings.md) ŌĆö dead button + refusal, not hidden slots.

- Separate **offer id per companion** (`hireScout`, `hireHealer`, `hireMule`, ŌĆ”) ŌĆö no `town.specialty` field.

**Rejected for v0.8:** hide empty hire slots; rotate one hire button; replace dead hire with next pool offer.

### Lore cycling (shipped)

- **Stable per modal visit:** denials, town/farm routine purchases (`buyFood`, `buyTroops`), hire refusals ŌĆö `encounterStableLine`.
- **Cursor advances (variety):** rumors, combat pay success / flee / victory ŌĆö `advanceCursor`.

### Preview plates

Each offer spec may define `previewPlate`. Encounter provider **concatenates** all visible offersŌĆÖ lines (town/farm/camp). Camp search: food + troops; hire: scout cost. **Deferred:** hover-to-focus one offer ŌĆö see backlog.

### Wyrm

Combat pay close outcome **`'paid'`** (union + `onWyrmCombatClosed`). Distinct from brigand ŌĆ£recruit via payŌĆØ loot path (same button, different fiction).

### Resource delta pops (`applyDeltas`)

- **Single path:** mechanics and reducer commit resource/message changes through `applyDeltas` in `encounterHelpers.ts`. Low-level `enqueueDeltas` stays in `uiAnim.ts` (implementation detail only).
- **Optional diff:** when `deltas` is omitted and `resources` is set, `resourceDeltasFromDiff(prev, next)` enqueues food/gold/army pops (healer mend after fight, town upkeep on enter, etc.).
- **Explicit lists** still used when one action needs multiple pops on the same target (e.g. pay ŌłÆgold then +loot); same-target stacking offsets horizontally in the renderer.
- Pops use `ui.clock.frame` at enqueue time (no special `startFrame` on the apply path). Grid enter-after-move-slide still schedules via `applyEnterAnims`.

### Status bar (party / inventory icons)

- TIC status strip loops `resources.party` and `resources.inventory`; sprites via `inventorySpriteId()` ŌĆö party/inventory ids match `SPRITES.inventory` keys (`beast`, `bloodVial`, `scout`, `healer`, ŌĆ”).
- **No mechanic list in `render.ts`** ŌĆö new companion adds party id + sprite row in `spriteIds.ts` only.
- Terminal preview plates: `terminalPlateLabel(spriteId)` derives labels from `SPRITES` keys (`army`, `hp` for troop/heart icons).

### Polish bundled in v0.8

- Player-visible lore: ASCII `-`, not em dash (learnings rule).
- Camp scout hire success: explicit `salt` on stable line; pool `CAMP_SCOUT_HIRE_LINES`.

## Acceptance Specs (GWT)

;===============================================================
; Healer mends after victory
;===============================================================
GIVEN the player has the healer and army size 10 at combat start.
WHEN the player loses 3 troops in rounds then wins.
THEN army size is 9.

;===============================================================
; Healer does not mend flee rear guard
;===============================================================
GIVEN the player has the healer and army size 5 at combat start.
WHEN the player loses 1 troop in rounds then flees.
THEN army size is 3.

;===============================================================
; Rumor cap
;===============================================================
GIVEN the player is in a town modal with rumors available.
WHEN the player buys three rumors in that visit.
THEN a fourth purchase spends no gold and is refused.

;===============================================================
; Economy offer guarantee
;===============================================================
GIVEN world seed S.
WHEN each town, camp, and farm is generated.
THEN each has at least one non-companion_hire offer.

;===============================================================
; Companion already
;===============================================================
GIVEN the player has a scout.
WHEN the player presses hire scout at a town that offers it.
THEN a companion-already line is shown and gold is unchanged.

;===============================================================
; Party full
;===============================================================
GIVEN the player party has three companions and no scout.
WHEN the player presses hire scout at a town that offers it.
THEN a party-full line is shown and gold is unchanged.

;===============================================================
; Healer upkeep on town enter
;===============================================================
GIVEN the player has the healer and 5 gold.
WHEN the player steps onto a town tile.
THEN gold is 4 before the town modal opens.

;===============================================================
; Pay exit without fighting ŌĆö no mend
;===============================================================
GIVEN the player has the healer and opens wyrm combat without fighting.
WHEN the player pays to end the encounter.
THEN army size is unchanged by healer mend.
