| Parting fiction | Voluntary — they are needed elsewhere, you need gold; never *sell people* copy ([`lore-and-tone.md`](../lore-and-tone.md)) |

| Boar volley beat | Volley-only press (no round RNG); Fight odds show **100%** when volley will fire |
| Mule↔boar hire refusal | `BOAR_MULE_REFUSED_LINES` when buying boar with mule held; `MULE_BOAR_REFUSED_LINES` when buying mule with boar held; **highlight held blocking slot** |
| Healer highlight | Mend on combat close when army rises; upkeep gold on town enter |
| TIC lore cart | Quote-parity `"` assignment at end of `lore.ts` (remove when TIC parser fixed) |
| Tile enter events | `TileEnterResult.events` for highlights on enter (e.g. healer upkeep) |
| Crossing last sell | Auto-close encounter when the last companion is sold |
| Duplicate hire refusal | Same — highlight **held** slot |
| Food carry full | Existing message + **highlight `army` stat** (they carry rations) |
| Mule cap saved | Highlight **`mule`** when a food gain from ≤ base cap lands higher than without the +50 bonus |
| No gold | Highlight **`gold` stat** on purchase/hire refusal (`feedbackChange` + `NO_GOLD_LINES`) |
| Mule hire success | Highlight **`food` stat** (carry cap just rose) |
| Boar volley beat | Highlight **`boar`** + enemy army delta on same Fight beat |
| Terminology | **Highlight / pulse** — not “flash”. Event: `iconHighlighted`; sprites: `slotHighlight0`…`3` |
| Party slot ids | `mule`, `boar`, `scout`, `healer` — stable in code; lore vocabulary may vary |
| Farm offers | `buildOfferSets`: `requiredOnEveryPoi` food; `mustCover` mule + boar; backfill skips duplicates (food already on row); 1–3 offers per farm |
| Must-cover placement | Each must-cover offer lands on a **random eligible PoI** (room, not duplicate); same for towns, camps, farms |

| Hire refusal order | Party full → already held → mule↔boar pairing → no gold (`feedbackChange` highlights gold) |
| Healer mend on recruit | Mend uses army **before** recruited enemy troops are counted; then recruits added back |
| Crossing sell lines | No fallback pool for unknown slot ids (`return {}`) |

## The Crossing

### Diegesis

A bridge or ford where caravans meet. Drovers and travellers **buy pack animals and hire willing companions** from passing companies. Mechanic is sell-only; fiction is bi-directional trade.

### Enter behaviour

```text
onEnterCrossing:
  if party.length === 0:
    return { message }  // CROSSING_EMPTY_LINES — no encounter
  else:
    roll sellGoldBySlot for each held slot (keyed RNG, cache on CrossingEncounter)
    open encounter modal
```

### Modal layout

- **Dynamic right grid:** one button per held party slot (sprite = `inventorySpriteId(slot)`), badges show `+Ng` from cached sell price.
- **Leave** on R (convention).
- Empty grid slots hidden — never show dead sell buttons for empty party indices.

### Sell reducer

- Remove slot from `party`; add cached gold.
- Success line from per-slot `*_SELL_LINES` (voluntary-parting register).
- `encounterClosed` outcome `'purchase'` or dedicated `'sell'` if union needs it — prefer reusing closest existing outcome for translator.

### Worldgen

- `CROSSING_COUNT`: 1–2 (constant in `constants.ts`).
- `place.crossing` stream; `CROSSING_NAME_POOL` in `lore.ts`.
- `poiSignpost.rank: 35` (between town 30 and camp 40).
- `mapLabel: 'X'`.
- `kinds: ['crossing']`; new `CrossingEncounter` + `EncounterKind` union entry.

### Illustration

- Left panel: `SPRITES.flavor.crossingRider` (172).
- Overworld / center preview: `SPRITES.poi.crossing` (44).

## Farm pool (mule + boar)

| Offer id | Category | Slot | Sprite |
|----------|----------|------|--------|
| `FARM_BUY_FOOD` | economy | — | food |
| `FARM_BUY_MULE` | companion_hire | `mule` | mule 226 |
| `FARM_BUY_BOAR` | companion_hire | `boar` | boar 228 |

```ts
buildOfferSets({
  poiCount: FARM_COUNT,
  pool: [buyFood, buyMule, buyBoar],
  requiredOnEveryPoi: [buyFood],
  mustCover: [buyMule, buyBoar],
  rng: RNG.createStreamRandomFromSeed(seed, 'farm.offers'),
})
```

Must-cover uses the same random eligible-PoI pass for towns, camps, and farms. `requiredOnEveryPoi` runs **before** must-cover so per-PoI staples (farm food) are never displaced by map-wide hire placement. Towns also benefit from pool (5) > `POI_MAX_OFFERS` (3). Farm backfill uses the full pool; food is skipped per-PoI because it is already on the row.

A farm may offer: food only, food+mule, food+boar, or food+both (3 buttons). `POI_MAX_OFFERS = 3` fits.

### Mule ↔ boar exclusion

At hire time, `refuseCompanionHire` pattern extended:

- Hold `mule`, press boar → `BOAR_MULE_REFUSED_LINES`, highlight **mule**.
- Hold `boar`, press mule → `MULE_BOAR_REFUSED_LINES`, highlight **boar**.

No runtime exclusion beyond hire — both may not be in `party` simultaneously.

## Boar — opening volley

`CombatEncounter` carries `boarVolleyFired: boolean` (false on open). On `reduceCombatFight`, **before** round resolution, when `party.includes('boar')` and `!boarVolleyFired`:

1. `kills = boarOpeningVolleyKills(encounter.enemyArmySize)`
2. Subtract from `enemyArmySize` (floor at 0); set `boarVolleyFired: true`.
3. Emit `resourceChanged` enemy delta + `iconHighlighted { band: 'party', id: 'boar' }`.
4. **Return** — volley is its own Fight press; normal rounds follow on later presses. If volley wipes the band, resolve victory without consuming fight RNG.

`combatFightHitOddsPercent` returns **100%** when the next Fight press will fire the volley.

Re-entering combat (new encounter) resets the volley. Lore: *"She'll do once, hard, at the start of any fight. Then sleep the rest off."*

Pure helper in `src/core/combat/` or `mechanics/defs/combat.ts`; one unit test table:

| enemy | kills |
|-------|-------|
| 8 | 3 |
| 20 | 5 |
| 40 | 10 |

## Mule — carry cap highlight

On food gain paths using `applyFoodCapOnGain`:

```text
baseCap = 2 * armySize
muleCap = foodCarryCap(resources)  // includes +50 when mule held
if mule held AND clampedAtMule > clampedAtBase AND food actually rose:
  emit iconHighlighted { band: 'party', id: 'mule' }
```

On buy-food refusal when `food >= foodCarryCap`:

- Keep `FOOD_CARRY_FULL_MESSAGE`
- Emit `iconHighlighted { band: 'stats', id: 'army' }`

## Icon highlight primitive

Extends [`2026-06-07-event-driven-animation-design.md`](./2026-06-07-event-driven-animation-design.md) Phase 5.

### Domain event

```ts
export type HighlightTarget =
  | { band: 'stats'; id: 'food' | 'gold' | 'army' }
  | { band: 'party'; id: string }
  | { band: 'inventory'; id: string }
  | { band: 'meta'; id: 'position' | 'oriented' | 'steps' }  // reserved; v0.9 may not emit

| { kind: 'iconHighlighted'; target: HighlightTarget }
```

`meta.position` highlights on lost teleport and on re-orient (`knowsPosition` false → true).

### TIC translator

- New `IconHighlightAnim`: 4 frames, ~300ms total, **non-blocking**.
- Cycles `SPRITES.ui.slotHighlight0`…`3`.
- Multiple highlights in one action batch may share `phaseCursor` (same frame).

### Render

- **Center** 16×16 pulse on target:
  - `party` / `inventory`: 16×16 held-band icon origin (same rect as `heldBandIconOrigin`).
  - `stats`: hero stat icon center in left panel.
- One code path: `highlightRectForTarget(target, layout)`.

## Post-ship: player feedback (2026-06)

**Considered:** A broader `feedback` event system unifying lore lines and highlights (purchase success/failure, tile enter, combat beats) with optional mechanic hooks or a central registry.

**Shipped instead:** Minimal slice only — `feedbackChange()` + `buy()` `shortfall` resource. Mechanics keep owning lore pools and RNG; core derives `iconHighlighted` from `reason` (gold/food shortfall). Wired for town, farm, camp hire, locksmith, combat pay `noFunds`.

**Deferred:** Carry-cap refusal, hire-blocked reasons, companion-effect highlights, optional `{ kind: 'feedback' }` domain event, lore-picker rules (stable per tile vs per action). Full design archived in [`backlog.md`](../backlog.md) § After demo → Player feedback.

**Rename:** `TOWN_NO_GOLD_LINES` → `NO_GOLD_LINES` — pool was always shared across PoIs; name was misleading.
